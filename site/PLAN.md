# 百泉古建筑群网站计划

## 已完成

### 站点结构

- 已完成 `React + TypeScript + Vite` 前端搭建
- 已建立内容同步脚本，从 `各建筑介绍/` 自动生成前端数据
- 已接入总览图、点位数据与 15 处建筑内容

### 浏览体验

- 首页已改成总览图点位进入，不再是普通列表
- 点位悬停、预览、点击入景与详情切换已经打通
- 详情页已改为独立阅读页面，去掉右侧冗长栏式布局
- 总览页到详情页已加入更顺滑的镜头推进感
- 预览卡与点位交互已修正闪烁抖动问题
- 总览图已做满屏首屏展示

### 风格方向

- 页面整体已从“开发中展示页”调整为更古风、简洁、沉浸的调性
- 已减少明显的开发说明、调试感和通用产品面板感
- 页面层级更偏“长卷 + 景中入景”，而不是后台式组件拼贴

### 导游系统

- 已加入灯笼式导游入口
- 已加入场景切换时的“耳语”提示，避免导游面板常驻
- 已加入路线化游法，不再只是普通问答
- 已加入行游札记区，导游更像陪伴式讲解
- 已把导游人设与规则拆到独立文件
- 已保留本地 `mock` 与真实 API 切换能力
- 已把导游模块改为懒加载，减轻首屏压力

## 当前目标

当前版本目标不是一次塞满所有功能，而是把下面三件事先做稳：

1. 首页总览图足够沉浸，进入建筑有“入景”感
2. 详情页阅读顺滑，不像资料后台
3. 导游体验像“同行引游”，而不是通用 AI 聊天框

## 下一步建议

### 第一优先级：继续提升沉浸感

1. 继续细修总览图热区形状与高光层次
2. 再加强总览页到详情页的转场节奏
3. 为重点建筑做更有差异的详情页首屏氛围
4. 进一步弱化“网页组件感”，让页面更像展陈与画卷之间的过渡

### 第二优先级：补导游专用内容

1. 为每个建筑补一段 80-150 字的导览词
2. 为每个建筑补亮点标签与图像说明词
3. 为四条路线补更精炼的导语和收束语
4. 让导游更会区分实景图、古风图、素描图

### 第三优先级：接真实 Agent

1. 后端接入 `Responses API`
2. 后端接入自己的导游人设文件
3. 后端按 `spotId`、`activeRouteId` 组织上下文
4. 先做好文本导游，再补 TTS 讲解

## 后续可扩展

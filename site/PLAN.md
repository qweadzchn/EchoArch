# 百泉古建筑群网站开发状态

## 当前建议

当前阶段先不要继续扩写“公共记忆回写”这一层。
现在更合适的策略是：

- 保持现有整体架构不再大改
- 继续做页面与交互的小修小磨
- 稳住导游体验、页面细节、视觉完成度
- 把“公共记忆回写、聚合、评估”明确放入下一阶段

一句话判断：
当前项目已经从“原型”进入“可展示成品打磨期”，不适合再突然开太多新系统。

## 目前已经完成的部分

### 1. 网站整体结构

- 已完成 `React + TypeScript + Vite` 前端架构
- 已形成多层页面结构，而不再是单页堆叠
- 已有首页、总览、景点详情、人物故事、书院历史、碑刻欣赏、导览页等主干页面
- 已支持二级、三级内容页跳转

### 2. 展示体验

- 首页已是全屏沉浸式首屏
- 总览页已支持地图式浏览与点位进入
- 景点详情页已改成更偏“章节阅读”的沉浸式结构
- 人物、碑刻、书院等内容已拆成独立章节，不再全部塞在同一层
- 页面滚动提示、章节跳转、回首页等主流程已经具备

### 3. 导游系统

- 前端已接入“泉上引路人”入口
- 已支持轻提示、展开面板、导览问答
- 已支持结构化动作
- 当前已支持的动作：
  - `open_spot`
  - `go_home`
  - `select_route`
- 已支持路线化游览而不是纯聊天
- 已支持本地导游后端 `server/guide-agent.ts`
- 已支持通过 `.env.guide` 接入兼容 OpenAI 风格的模型接口
- 已兼容 `responses` 和 `chat.completions` 两类接口风格

### 4. 导游资料能力

- 已把项目内部景点资料接入导游检索链路
- 已完成内部资料分块检索的基础版本
- 已完成公共记忆的“读取侧”基础结构
- 已有以下公共记忆骨架：
  - `scene_hint_profiles`
  - `hot_questions_by_spot`
  - `common_confusions`
  - `route_acceptance_patterns`
  - `blind_spots`

### 5. 体验补充

- 已加入环境音乐入口，并改成低存在感样式
- 已有导览配置文件、系统设计文档、公共记忆设计文档

## 目前已经打底，但还不该继续猛做的部分

这些部分已经有基础，但现在更适合“保留、记录、后续继续”，而不是马上深挖：

- 公共记忆回写
- 游客行为事件采集
- 批量聚合与记忆升级
- 更细的知识图谱关系层
- 语音导游闭环

也就是说：
现在“会读共享经验”的地基已经有了，
但“把游客行为自动沉淀成共享经验”的闭环还没有开始正式做。

## 当前项目最适合继续做的事情

### 第一优先级：成品细节打磨

- 修正文案换行、卡片挤压、局部遮挡、对齐不齐等小问题
- 统一不同页面之间的留白、按钮尺寸、圆角、信息密度
- 继续降低“网页感”和“开发中痕迹”
- 让页面更像一个完整作品，而不是若干模块拼起来

### 第二优先级：导游体验打磨

- 让导游更像陪游，不像聊天框
- 优化导游的出现时机、收起时机、行动反馈
- 继续控制导游的打扰感
- 继续强化“导游带人走”的感觉，而不只是回答问题

### 第三优先级：内容层细修

- 补齐少数页面里仍偏空、偏松、偏散的版面
- 优化部分景点内容的阅读节奏
- 继续按 PPT 要求微调章节结构与表达方式

## 下一阶段开发

### Phase A：成品打磨

- 继续修 UI 细节与响应式问题
- 收紧页面节奏、减少破碎感
- 完善导游面板和引导反馈的小动作

### Phase B：导游稳定化

- 继续优化 API 接入稳定性
- 提升导游响应一致性
- 进一步打磨动作触发边界，避免误跳转

### Phase C：公共记忆闭环

这一阶段再开始，不提前。

目标：

- 记录匿名游客事件
- 在景点闭环、路线闭环、会话结束时生成候选记忆
- 只把高置信、可复用、低噪声的模式升级为公共记忆

当前判断标准：

- 先记事件，不急着记结论
- 先做候选，不急着写入长期记忆
- 先做聚合，不做逐轮即时回写

### Phase D：更深的智能导览能力

- 更细粒度的知识关系组织
- 热门问题缓存与预取
- 语音讲解
- 更丰富的导览动作系统

## 当前项目状态总结

如果要一句话概括现在的网站状态：

它已经不是“功能原型”，而是一个已经具备展示、讲解、跳转、路线、内容分层能力的可用成品；
现在最需要的是继续把它磨顺、磨稳、磨高级，而不是立刻再开一层复杂系统。

## 这轮决定

本轮先采取这个策略：

- 保持当前导游与公共记忆读取能力
- 暂不继续推进公共记忆回写实现
- 优先修视觉和交互小细节
- 公共记忆回写明确放入下一阶段开发

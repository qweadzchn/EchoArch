# 百泉古建筑群网站计划

## 当前已完成

### 站点结构

- 已使用 `React + TypeScript + Vite` 搭建前端项目
- 已建立素材同步脚本，将外层 `各建筑介绍` 自动生成前端数据
- 已接入 `首页.jpg` 与 `首页和建筑点位对照.jpg`
- 已将 15 处建筑点位映射到首页总览图

### 浏览方式

- 首页已改为总览图热区进入
- 点击点位后进入独立建筑详情页
- 已去除右侧长栏式阅读方式
- 已修复热区悬停时预览卡导致的闪动问题
- 首页主图已改为全屏式首屏展示
- 预览卡已改为仅在悬停时出现，不再常驻遮挡画面
- 热区框已改为默认隐藏，仅保留编号印记做引导，悬停时再显现

### 视觉方向

- 当前基调已转为古风、克制、偏沉浸式
- 已减少页面中的开发说明和“AI 味”文案
- 已将过程信息从页面中移出，避免破坏观感
- 首页热区已扩大实际悬停感应区，编号与边缘附近的交互更稳定
- 预览卡已改为常驻节点做柔和显隐与滑移，不再频繁重挂载造成卡顿感
- 已收敛部分过重的阴影、模糊和描边，让首页更像画卷而不是组件拼贴

## 当前版本目标

当前版本的目标不是把所有功能一次堆满，而是先把基础体验定稳：

1. 首页要有“看一幅长卷”的感觉
2. 进入建筑详情页要足够顺滑
3. 图文阅读不能再像后台面板或产品介绍页
4. 后续接入导游智能体时，不需要推翻现有结构

## 下一步迭代建议

### 第一优先级：继续提升沉浸感

1. 给首页加更细腻的热区动效
2. 加强从总览进入详情页的转场
3. 提升详情页的节奏感，让首屏更像展陈而不是资料页
4. 根据每个建筑的气质，细调主图、色调和标题排版
5. 继续减少“网页组件感”，让页面更像长卷与展板之间的过渡形态

### 第二优先级：增强内容表现

1. 将部分长文案改写为更适合网页阅读的分段摘要
2. 为每个建筑补一段简短导览词
3. 区分实景图、古风图、素描图等不同素材类型
4. 为重要建筑补更有层次的图像编排方式

### 第三优先级：导游智能体接入准备

1. 设计导游入口的位置与出现方式
2. 确定导游模式是悬浮角色、侧边对话还是语音讲解
3. 建立点位上下文结构
4. 让模型能根据当前建筑、相邻建筑和游览路径生成讲解内容

## 后续可做的增强项

- 真实热区轮廓替代矩形框
- 点位进入时的镜头推进动画
- 背景环境音或水声、风声
- 建筑间的推荐游线
- 白天 / 黄昏两套视觉氛围
- 面向大屏展示的自动导览模式
